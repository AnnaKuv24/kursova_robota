## Технологічний стек

### Frontend
- **React.js** (v18) — основна UI-бібліотека
- **React Router DOM** (v6) — клієнтська маршрутизація
- **Bootstrap** (v5.3) — CSS-фреймворк для адаптивного макету
- **HTML5 / CSS3** — Flexbox, Grid, Media Queries
- **`src/data.js`** — джерело даних порталу (новини, галерея, інформація про університет, контакти)

### Хостинг
- **Netlify** — деплой статичної збірки React (без серверної частини)
- **Netlify Forms** — обробка форми зворотного зв'язку без бекенду

### Архів (попередня архітектура)
До переходу на статичний хостинг дані віддавались через окремий бекенд:
- **Node.js + Express.js** — REST API
- **JSON (db.json)** — файл-база даних

Цей варіант залишено у папці `server/` для довідки, але він **не використовується** у поточній (Netlify) версії порталу.

## Джерело даних (client/src/data.js)

Усі дані порталу зберігаються прямо у клієнтському коді у файлі `client/src/data.js`:

- `newsData` — масив новин (id, title, summary, content, date, image, category)
- `galleryData` — масив фото галереї (id, title, url, category)
- `aboutData` — інформація про університет (опис, місія, статистика, факультети)
- `contactsData` — контактні дані (адреса, телефон, email, графік роботи)

Щоб змінити вміст сайту (додати новину, фото, оновити контакти тощо), потрібно відредагувати цей файл і задеплоїти зміни (`git push` → Netlify перебілдить сайт автоматично).

## Функціонал порталу

### Головна сторінка (/)
- Шапка сайту з логотипом та адаптивним меню
- Автоматичний слайдер з 3 слайдами (5 секунд / слайд)
- Блок статистики університету
- Секція «Про університет»
- Вивід 10 останніх новин
- Заклик до дії (CTA)
- Підвал з контактами та навігацією

### Про сайт (/about)
- Загальна інформація про університет
- Статистика у цифрах
- Перелік факультетів

### Галерея (/gallery)
- Сітка фотографій (3 колонки)
- Фільтрація за категоріями
- Lightbox для збільшення зображень

### Новини (/news)
- Список всіх новин (картки)
- Пошук по заголовку та тексту
- Фільтрація за категоріями
- Детальна сторінка новини (/news/:id)

### Контакти (/contacts)
- Контактна інформація (адреса, телефон, email)
- Вбудована карта Google Maps
- Форма зворотного зв'язку з валідацією (відправка через Netlify Forms)

## Запуск проекту локально

### Вимоги
- Node.js v16 або вище
- npm v7 або вище

```bash
cd client
npm install
npm start
# Додаток відкриється на: http://localhost:3000
```

Серверну частину (`server/`) запускати не потрібно — клієнт повністю самодостатній і не звертається до жодного API.

### Перевірка production-збірки
```bash
cd client
npm run build
# Результат — папка client/build, готова для деплою
```

## Деплой на Netlify

1. Запушити проект у GitHub-репозиторій
2. На [netlify.com](https://netlify.com): **Add new site → Import an existing project** → обрати репозиторій
3. Налаштування збірки:
   - **Base directory:** `client`
   - **Build command:** `npm run build`
   - **Publish directory:** `client/build` (або `build`, якщо base directory вже `client`)
4. Після підключення кожен `git push` автоматично запускає новий деплой

### Форма зворотного зв'язку
Форма на сторінці «Контакти» відправляється через **Netlify Forms** і не потребує бекенду. Прихована форма-«шаблон» для виявлення Netlify знаходиться в `client/public/index.html`. Після першого деплою на Netlify повідомлення з'являтимуться у **Site settings → Forms**.
